제 8 회 "ACCESS를 이용한 데이터베이스 구축 및 활용 실습" 강의는주제상 "제 7 회 데이터베이스 설계 실습"의 연속입니다. "제 7 회 데이터베이스 설계 실습"에서 도출된 릴레이션 스키마를 중심으로 다음 내용을 파악하면서 강의에 임하길 바랍니다.

  • 데이터베이스를 생성하기 위한 단계를 알아 봅시다.
  • ACCESS에서 릴레이션을 어떻게 생성하는지를 알아봅시다.
  • ACCESS에서 참조무결성은 어떻게 정의하는지를 알아봅시다.
  • QBE를 이용한 질의 작성이 어떻게 행해지는지 알아 봅시다.
  • QBE와 SQL이 어떻게 대응되는지 알아봅시다.
  • 작성된 "강의시스템" ACCESS 데이터베이스가 압축되어있으니 다운로드 받기 바랍니다.

      본 강의에서는 '제 7 회 데이터베이스 설계 실습'에서 최종적으로 도출된 데이터베이스 스키마를 중심으로 ACCESS97을 이용하여 데이터베이스를 구축하고 활용하는 내용을 다룬다. 따라서, 수강생 여러분들은 본 강의가 제 7 회 강의 내용의 연속임을 명심하길 바란다.

      데이터베이스 구축을 실습하기 위해  ACCESS를 이용하는 이유는 ACCESS는 다른 데이터베이스 시스템에 비해 PC 상에서 쉽게 이용할 수 있고 배우기 쉽기 때문이며, 따라서, 데이터베이스 구축에 대하여 수강생 여러분들의 이해를 빨리 돕도록 하기 위함이다. 사실 상, 데이터베이스 구축과 활용 방법은 DBMS마다 각각 다르므로 사용방법에 대한 매뉴얼을 반드시 참고해야 한다.

2. 데이터베이스 구축 과정

      일반적으로 데이터베이스는 다음의 절차를 거침으로써 구축된다.

       

      1. 데이터베이스 명칭 부여 및 디스크 영역 할당

        데이터베이스 명칭을 부여하고 데이터베이스가 저장될 디스크 영역의 크기 등을 정의하는 단계이다.

         

      2. 릴레이션 생성

        릴레이션 생성은 릴레이션을 구성하는 속성들과 각 속성의 타입(도메인), 주키이 설정, 외래 키이 설정, 제약조건 등을 차례로 선언함으로써 이루어진다.

         

      3. 튜플 삽입

        각 릴레이션에 해당 튜플들을 삽입하는 단계이다.

      이 절차에 따라 ACCESS를 이용하여 데이터베이스를 구축하도록 하자.

    2.1 데이터베이스의 명칭 부여

      ACCESS의 초기 화면

      ACCESS를 실행하면 다음의 그림 8-1과 같은 초기 윈도우 화면이 생성되고 데이터베이스를 구축할 수 있는 환경이 조성된다.

       

      그림 8-1. ACCESS의 초기 윈도우 화면

       

       위의 그림 8-1 초기 화면에서 파일 메뉴를 선택한 후 새 데이터베이스 부 메뉴를 선택하거나 아니면 아이콘 을 선택하면 다음과 같은 그림 8-2의 화면이 열린다.

       

      그림 8-2. 새 데이터베이스 화면

       

      위의 윈도우 화면에서 확인 버튼을 누르면 생성할 데이터베이스의 명칭을 부여할 수 있는 다음 그림 8-3의 윈도우 화면이 생성된다.

 

그림 8-3. 새 데이터베이스 명칭 부여 화면

 

      위 그림 8-3의 윈도우 화면에서 파일 이름 항목에 강의시스템으로 부여하기로 하자. 데이터베이스 명칭을 부여한 후 작성 버튼을 누름으로써 릴레이션이 없는 데이터베이스가 생성되고 디스크 상에 영역이 할당된다.

    2.2  릴레이션 테이블의 생성

      그림 8-3에서 작성 버튼을 누르면 다음 그림 8-4와 같이 릴레이션(테이블)을 생성할 수 있는 윈도우가 열린다.

       

      그림 8-4. 릴레이션(테이블), 질의, 폼 등의 생성 화면

       

      위의 그림 8-4는 ACCESS 데이터베이스 구성요소를 생성할 수 있는 윈도우 화면을 나타낸다. 그림 8-4에서 표시된 메뉴의 종류와 기능을 설명하면 다음과 같다.

      • 테이블 : 릴레이션을 생성하고 관리하는 기능 메뉴이다
      • 질의 : 질의를 생성하고 관리하는 기능 메뉴이다
      • 폼 : 윈도우 형태로 데이터베이스를 액세스하고 관리할 수 있도록 하는 사용자 인터페이스를 생성하고 관리하는 기능이다.
      • 보고서 : 데이터베이스 질의 결과를 출력하기 위한 형식을 정의하고 관리하는 기능이다.
      • 매크로 : 폼과 관련하여 수행되는 연산 매크로를 정의하고 관리하는 기능이다.
      • 모듈 : 폼과 관련하여 수행되는 프로시져를 프로그래밍하고 관리하는 기능이다.

      그림 8-4에서 테이블 메뉴를 선택하고 새로만들기 메뉴를 선택하면 다음 그림8-5와 같은 메뉴가 제시된다.

       

      그림 8-5. 릴레이션(테이블) 구축을 위한 메뉴

       

      위의 그림 8-5에서 디자인 보기 메뉴를 선택해 보자. 이 메뉴를 선택하면 다음 그림 8-6과 같이 릴레이션(테이블)의 속성(필드)과 속성의 도메인(데이터 형식)을 정의할 수 있는 화면이 열린다.

       

      그림 8-6. 릴레이션(테이블) 생성 화면

        

      그림 8-6에서 필드 이름 난은 속성 명칭을 입력하는 난이며 데이터 형식 난은 속성의 도메인을 입력하는 난이다.  그림 8-6은 표 8-1의 학과 릴레이션의 속성 학과번호, 학과명,단대번호를 정의하고 있는 과정이다. 학과번호는 주키이 이므로 주키이 선언을 해야 한다. 주키이 선언은 해당 속성을 선택한 후 마우스의 오른쪽 버튼을 누르면 메뉴들이 나오는데 그 메뉴 중에서 기본키이 메뉴를 선택함으로써 이루어진다. 여러 속성으로 이루어진 주키이는 ctrl 키이를 누른 상태에서 주키이 속성들을 선택하고 ctrl을 누른 상태에서 마우스 오른쪽 버튼을 눌러 메뉴가 돌출되게 한 후 기본키이 메뉴를 누름으로써 선언된다. 주키이로 선언된 속성(필드)은 그림 8-6의 학과번호 속성 오른쪽처럼 열쇠가 표시된다.

      릴레이션의 모든 속성(필드)들과 속성 도메인, 그리고 주키이를 정의한 뒤 저장 아이콘()을 누르면 테이블(릴레이션)의 이름을 입력하는 메뉴가 돌출되며 이때 릴레이션의 이름인 학과를 입력함으로써 다음 그림 8-7과 같이 어떤 릴레이션(테이블)이 생성되었는 지를 나타내는 화면으로 복귀한다.

       

      그림 8-7. 릴레이션(테이블)의 생성을 나타내는 화면

          

      다음 그림 8-8은 두 개 이상의 속성으로 이루어진  주키이를 선언하는 예를 나타낸다.

       

      그림 8-8. 두 개이상의 속성을 주키이로 선언된 예

       

      표 8-1의 나머지 각 릴레이션 스키마에 대해 그림 8-4부터 그림 8-7까지의 과정을 적용함으로써 다음 그림 8-9와 같이 모든 릴레이션을 생성 할 수 있다.

        

      그림 8-9. 표8-1의 모든 릴레이션의 생성 화면

          

       위 그림 8-9의 생성된 릴레이션은 튜플을 갖지 않은 상태이다. 그림 8-9의 각  릴레이션들은 SQL의 CREATE 명령에 의해 속성과 속성 도메인, 그리고 주키이 선언을 포함하여 릴레이션을 생성한 것과 같다. 그러나, 외래 키이 선언은 포함되자 않은 상태임을 주시하자.

    2.3 참조무결성(외래 키이) 선언 

      ACCESS에서 외래 키이 선언은 참조무결성(referential integrity)을 선언함으로써 이루어진다. 참조 무결성은 그림 8-10와 같이 주 메뉴의 도구 메뉴에서 관계(R) 부 메뉴를 선택함으로써 시작된다.

       

      그림 8-10. 참조무결성 선언을 위한 관계(R) 선택 화면

       

       관계(R) 메뉴를 선택한 후 마우스 오른쪽을 누르면 다음 그림 8-11같이 작은 메뉴 돌출된다.

       

      그림 8-11. 참조무결성 선언을 돌출 메뉴

            

      위의 그림 8-11에서 테이블 표시 메뉴를 선택하면 다음 그림 8-12와 같은 테이블 표시 메뉴가 돌출된다.

       

      그림 8-12. 참조무결성에 관련된 릴레이션(테이블) 선택 화면

        

      표 8-12의 테이블 표시 메뉴에서 참조무결성에 관련된 릴레이션(테이블)들을 하나씩 선택하여 우측의 추가 버튼을 차례대로 선택한 후 닫기 버튼을 누름으로써 참조 무결성에 관련된 선택된 모든 릴레이션(테이블)들이 다음 그림 8-13과 같이 나열된다.

       

      그림 8-13. 참조무결성 관계에 선택된 릴레이션(테이블)들

                

      위의 그림 8-13에서 참조 무결성 관계를 부여하는 한 예를 들어보자. 표 8-1에서 강좌요일 릴레이션에서 강좌번호 속성은 외래 키이로서 강좌 릴레이션의 강좌번호 속성을 참조한다. 이때 참조 릴레이션인 강좌 릴레이션의 강좌번호 속성을 선택하여 마우스 왼쪽을 누른 상태에서 강좌요일 릴레이션의 강좌번호 속성으로 마우스 끌기를 실행하면 다음 그림 8-14와 같이 관계 메뉴가 돌출된다.

       

      그림 8-14. 참조무결성 선언

 4. 질의 생성

      ACCESS에서 질의는 QBE(Query By Example)를 이용하여 작성뙨다. QBE에서 질의는 테이블 형태로 표현된다. ACCESS에서는 QBE로 표현된 질의는 SQL로 대응되며 대응된 SQL은 ACCESS 엔진에 의해 처리된다. 본 강의에서는 질의를 죠인 연산이 없는 질의와 죠인 연산이 있는 질의를 각각 나누어 QBE로 표현한 예와 표현된 QBE 질의에 대응하는 SQL을 예로 보이기로 한다.

    4.2 죠인 연산을 포함하는 질의 생성

      질의-1은 죠인(join)을 포함하지 않는 연산이다. 죠인을 포함하는 연산은 두 개이상의 릴레이션이 이용된다. 죠인 연산을 포함하는 질의를 위해 다음 질의-2를 고려해보자.

      질의-2

      "운영체제 교과목을 수강하는 학생을 찾아서 강좌번호, 과목번호, 과목명, 학번, 이름을 출력하라"

      질의 생성, 저장, 실행

      질의-2에 대한 질의를 실행하기 위해서는 운영체제 교과목으로 개설된 강좌를 찾아서 이 강좌를 수강하는 학생을 찾으면 된다.

      이를 처리하는 SQL 질의를 작성하면 다음과 같다.

              

      표 8-2. 질의-2에 대한 SQL

      SELECT DISTINCT 강좌.강좌번호, 강좌.과목번호, 교과목.과목명, 수강.학번,

              학생.이름

      FROM 학생, 교과목, 강좌, 수강

      WHERE 교과목.과목명='운영체제' AND 강좌.과목번호=교과목.과목번호 AND

          강좌.강좌번호=수강.강좌번호 AND 학생.학번=수강.학번;
